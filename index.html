git-svn更换服务器域名

  1. 更换地址

    更换完域名后测试已不能更新

    xinlong @ localhost in ~/Documents/GIT/svntest/svnClient/xinzhu-test3/xinzhu-doc on git:master o [10:33:43]

  2. 切换到git-svn(git 默认给svn创建的版本)
    ``` bash
    xinlong @ localhost in ~/Documents/GIT/svntest/svnClient/xinzhu-test3/xinzhu-doc on git:master o [10:33:55] C:1
    $ git co git-svn
    Note: checking out ‘git-svn’.

You are in ‘detached HEAD’ state. You can look around, make experimental
changes and commit them, and you can discard any commits you make in this
state without impacting any branches by performing another checkout.

If you want to create a new branch to retain commits you create, you may
do so (now or later) by using -b with the checkout command again. Example:

git checkout -b

HEAD is now at 8b681cd… svn update2

3. 此时需要在svn主库上做一个提交,可以让其他用户随便修改或添加一个提交
4. 等其他用户提交完后我们在git-svn分支做一个更新
``` bash
 xinlong @ localhost in ~/Documents/GIT/svntest/svnClient/xinzhu-test3/xinzhu-doc on git:8b681cd o [10:33:58]
$ git svn fetch
    M   svntest
r81 = 50c7c8c44ba3a2a660da932067e6394ccf476917 (refs/remotes/git-svn)

# xinlong @ localhost in ~/Documents/GIT/svntest/svnClient/xinzhu-test3/xinzhu-doc on git:8b681cd o [10:35:45]
  1. 切换到master
    $ git co master
Switched to branch 'master'
  2. 执行 git rebase git-svn

    xinlong @ localhost in ~/Documents/GIT/svntest/svnClient/xinzhu-test3/xinzhu-doc on git:master o [10:35:51]

    $ git rebase git-svn
    First, rewinding head to replay your work on top of it…
    Fast-forwarded master to git-svn.
  3. 查看日志已经是最新的了

    xinlong @ localhost in ~/Documents/GIT/svntest/svnClient/xinzhu-test3/xinzhu-doc on git:master o [10:35:58]

    $ git one
  4. 执行 rebase已经不再报错了

    xinlong @ localhost in ~/Documents/GIT/svntest/svnClient/xinzhu-test3/xinzhu-doc on git:master o [10:36:07]

    $ git svn rebase
    Current branch master is up to date.
